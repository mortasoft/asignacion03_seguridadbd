01 — Concepto

¿Qué es SQL Injection?

SQL Injection ocurre cuando datos del usuario se insertan directamente en una consulta SQL sin validación. Un atacante puede alterar la lógica de la consulta para extraer, modificar o eliminar datos.

☠️

Versión Vulnerable (esta)

El parámetro apellido se concatena directamente al SQL. El atacante puede inyectar cualquier código SQL y el servidor lo ejecutará.

🔒

Versión Segura (v2)

Usa consultas parametrizadas con ? como marcador. Los datos nunca se mezclan con el código SQL — el driver los escapa automáticamente.

Python — Comparación de Queries 
# VULNERABLE (v1) — concatenación directa
query = "SELECT name FROM sys.databases WHERE name LIKE '" + apellido + "'"

# SEGURA (v2) — consulta parametrizada
query = "SELECT name FROM sys.databases WHERE name LIKE ?"
cursor.execute(query, (apellido,))
02 — Mecánica

Cómo Funciona el Ataque

Cuando el usuario ingresa un valor normal, la query filtra por apellido. Cuando inyecta SQL, altera la lógica completa de la consulta.

1

Consulta Normal

El usuario envía master. La query resultante filtra correctamente:

SELECT name, database_id FROM sys.databases
WHERE name LIKE 'master'
2

Payload de Inyección

El atacante envía ' OR '1'='1'--. La query resultante devuelve TODOS los registros:

SELECT name, database_id FROM sys.databases
WHERE name LIKE '' OR '1'='1'--'
--           ↑ siempre verdadero  ↑ comenta el resto
3

Resultado del Ataque

El servidor retorna todas las bases de datos del sistema, ignorando el filtro original. El atacante obtiene información que no debería ver.

03 — Demo

Prueba los Ataques

Haz clic en "Probar" para abrir el endpoint en una nueva pestaña y ver el resultado real. La respuesta incluye la query_ejecutada para que puedas ver el SQL generado.

Consulta Normal
Búsqueda legítima de bases de datos que empiezan con "m".
GET http://localhost:5001/cliente?apellido=m
↗ Probar Resultado esperado: master, model, msdb
Ataque — OR 1=1 (Bypass de filtro)
Inyecta una condición siempre verdadera para saltarse el filtro y listar TODAS las bases de datos del servidor.
GET http://localhost:5001/cliente?apellido=' OR '1'='1'--
↗ Probar Peligro: devuelve TODAS las DBs
Ataque — UNION SELECT (Extracción de datos)
Usa UNION para combinar el resultado con otra consulta SQL, extrayendo información de otras tablas del sistema.
GET http://localhost:5001/cliente?apellido=x' UNION SELECT name, database_id FROM sys.databases--
↗ Probar Peligro: UNION injection activo
¿Cómo se ve en la Versión Segura?

Los mismos payloads son tratados como texto literal en v2. El resultado siempre es [] porque no existe una base de datos llamada literalmente ' OR '1'='1'--.
